    • 搞清楚算法复杂度

    三种符号：

    Θ : theta, 西塔

    Ο : omikron, 奥秘可容

    Ω : omiga, 哦米噶

    theta(g(n)) 表示一个集合：

    存在正常量c1, c2 和 n0，使得所有n>= n0， 有 0 <= c1g(n) <= f(n) <= c2g(n)

    即theta表示一个渐进函数，使得目标函数可以使用常数c1,c2夹逼在渐进函数中。

    omikron 存在常数和函数，c,n0和g，则表示目标函数的上界函数，即目标函数f(n) <= cg(n), n >= n0

    omiga 存在常数和函数，c,n0和g，则表示目标函数的下界函数，即目标函数f(n) >= cg(n), n >= n0

    等式和不等式的渐进符号

    ο: 小omikron, 定义与大omikron相似，区别在于，对于任意常数c，都有0 <= f(n) < cg(n)

    ω: 小omiga, 定义相似，区别在于，对于任意常数c，都有 0<= cg(n) < f(n)

    分析算法

    最坏情况： 该算法的算法时间的一个上界。
    平均情况： 算法运行时间的平均情况。

    我们在非概率影响的算法，往往讨论它的最坏情况，因为它的衡量效果最好。

    对于两个算法来讲，我们往往用它的增长速度来衡量算法的好坏。

    例如 theta(n3) 比 theta(n2) 差。

    例子

    1. 对于插入排序的最坏情况有表达式

    T(n) = an**2 + bn + c

    则theta(T(n)) = theta(n**2)

    1. 对于递归算法

    (1) 构造递归树来分析算法：

    构造递归树，发现树的高度为lgn, 每层消耗cn, 加上根节点消耗， 就有总消耗

    T(n) = cnlgn + cn, 即 theta(nlgn)

    (2) 利用主方法求解

    求解 T(n) = aT(n / b) + f(n) 形式的递归函数 a >= 1, b > 1

    1.对于某个常数e > 0, 有f(n) = O(n logb(a - e)), 则 T(n) = O(nlogb(a))

    1. 若f(n) = theta(nlogb(a)), 则 T(n) = theta(nlogb(a) * lgn)

    2. 若对e>0,有f(n) = omiga(n**logb(a+e)), 且对某个常数c<1和足够大的n有af(n / b) <= cf(n), 则T(n) = theta(f(n))
